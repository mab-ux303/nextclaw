# v0.13.61-config-meta-opinionated-proactive-advice

## 迭代完成说明（改了什么）
- 调整 `AGENTS.md` 中 `infer-intent-and-be-proactive` 规则，增强为“必须给出明确立场与判断（推荐方案 + 取舍理由）”，不再只停留在复述与机械执行。
- 明确要求在回复结构中包含：`深层意图判断`、`我的判断/推荐`、`理由`、`最小可执行方案`、`可选后续动作`。
- 增加风险前置机制：当用户方案存在明显风险时，需先提示风险并提供更优替代路径。

## 用户/产品视角的验收步骤
1. 提出一个含糊或带取舍的问题（例如“这个需求直接做就行吗？”）。
2. 检查助手回复是否先给出深层意图判断，而不是只复述问题。
3. 检查回复是否明确表达推荐方案与理由，并给出可执行下一步。
4. 检查在明显风险场景下，助手是否会主动提示风险并给替代方案。
